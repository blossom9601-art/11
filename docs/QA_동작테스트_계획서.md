# Blossom (Lumina) 엔터프라이즈 자산관리 시스템 — QA 동작테스트 계획서

> **작성일**: 2025-04-11  
> **대상 시스템**: Blossom 플랫폼 / Lumina 서비스  
> **아키텍처**: WEB / AP(Flask) / DB(SQLite·MySQL) 3-Tier  
> **범위**: 전체 메뉴 페이지 이동, 기능 동작, 예외처리, 권한, 데이터 정합성, 사용성

---

## 1. 전체 테스트 전략

### 1-1. 테스트 관점

| 관점 | 설명 |
|------|------|
| **기능 정합성** | 각 메뉴/페이지가 CRUD 동작을 정상 수행하는가 |
| **페이지 이동 연속성** | 목록→상세→수정→저장→복귀 흐름에서 데이터·상태가 유지되는가 |
| **권한 격리** | ADMIN / 일반사용자 / READ 전용 / NONE 역할별로 메뉴·버튼·API가 정확히 차단되는가 |
| **세션 라이프사이클** | 로그인→하트비트→세션만료→동시접속 정책이 정상 작동하는가 |
| **예외 견고성** | 빈값, 특수문자, 중복클릭, 네트워크 오류 시 시스템이 안전하게 처리하는가 |
| **데이터 동기화** | 화면 표시값 ↔ API 응답 ↔ DB 저장값이 일치하는가 |

### 1-2. 핵심 흐름 (최우선 확인)

1. 로그인 → 대시보드 진입 → 메뉴 권한별 노출 확인
2. 하드웨어 자산 목록 → 상세 → 탭(01~15) 조회/등록/수정/삭제 → 목록 복귀
3. 프로젝트 목록 → 상세 → 탭(81~90) CRUD → 비용·일정·이해관계자 연계
4. 거버넌스(백업정책·취약점·패키지) 대시보드 → 상세 조회 → 정책 등록/수정
5. 카테고리(제조사·유지보수·부서·고객사) 등록/수정 → 자산 상세 드롭다운 반영

### 1-3. 운영장애 고위험 구간

| 구간 | 위험 요소 |
|------|-----------|
| 로그인 ~ MFA 인증 | 계정 잠금 오작동, MFA 코드 만료, IP 제한 우회 |
| 자산 상세 탭 CRUD | `blsMakeTabCrud` 프레임워크 버그 시 71~90 탭 전체 영향 |
| 대량 삭제 (bulk-delete) | soft-delete 후 참조 데이터 미정리, 삭제 후 상세 진입 |
| 동시접속 세션 정책 | `max_sessions=1` 시 기존 세션 강제 종료 타이밍 |
| 권한 변경 즉시 반영 | `_perms` 캐시가 세션에 저장되므로 변경 후 재로그인 필요 여부 |
| SSE/실시간 알림 | 네트워크 단절 시 이벤트 누락, 재접속 로직 |

#### TC-AUTH-003

- **테스트 ID**: TC-AUTH-003
- **대분류**: 공통
- **중분류**: 로그인
- **테스트명**: 5회 비밀번호 오류 → 계정 잠금
- **목적**: 로그인 5회 연속 실패 시 30분 계정 잠금 확인
- **사전조건**: 활성 계정, `login_fail_cnt=0`
- **수행절차**:
  1) 잘못된 비밀번호로 5회 연속 로그인 시도
  2) 6번째 시도: 올바른 비밀번호 입력
- **입력값 예시**: 비밀번호=`Wrong1`~`Wrong5` (5회), 이후 올바른 비밀번호
- **기대결과**:
  - 5회째: "계정이 잠겼습니다. 30분 후 다시 시도해 주세요" 메시지
  - 6번째(올바른 비밀번호): 여전히 잠금 메시지 표시, 로그인 불가
  - `locked_until` 필드에 현재시간+30분 설정
- **추가 확인 포인트**: 잠금 해제 후(30분 경과) 정상 로그인 가능한지, `login_fail_cnt` 초기화 여부
- **실패 시 의심 원인**: `locked_until` 타임존 불일치(UTC vs local), 잠금 검사 로직 순서 오류
- **우선순위**: P1
