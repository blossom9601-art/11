# AI 브리핑 3단계 계획 (규칙 기반 고도화)

## 범위
- 목표: 브리핑의 추천 필터, 근거 문장, 신뢰도 점수를 도메인 규칙으로 고도화
- 대상: _build_unified_search_briefing() 함수 강화
- 기대효과: 사용자 가이드 질 향상, 폴백 빈도 감소

## A. 추천 필터 규칙 (recommended_filters)
### 규칙 1: 도메인별 필터 추천 (쿼리 키워드 분석)
- 쿼리에 도메인 접두사 포함 시 해당 도메인 필터 추천
  - '서버' → '데이터센터' 필터 추천
  - '프로젝트' → '프로젝트' 필터 추천
  - '비용' → '비용관리' 필터 추천
  - '블로그' → '인사이트' 필터 추천

### 규칙 2: 결과 도메인 분포 기반 필터 추천
- 결과에 특정 도메인 집중 시 다양화 필터 제안
  - 동일 도메인 70% 이상 → "다른 카테고리도 검색하시겠어요?"

### 규칙 3: 타입별 필터 추천
- 결과 타입이 단일 타입 90% 이상 → 타입 다양화 필터
  - 예: '페이지만 나옵니다. 자산도 검색해보세요?'

## B. 근거 문장 강화 (summary_lines)
### 규칙 1: 쿼리 특성 분석
- 정확 쿼리 (따옴표 포함 또는 완전 일치): "정확도 높게 검색되었습니다"
- 부분 매칭: "유사한 항목들을 함께 표시했습니다"
- 와일드카드/정규식: "패턴 기반으로 검색했습니다"

### 규칙 2: 결과 특성 기반 근거
- total=0: "해당 결과를 찾지 못했습니다. 필터나 키워드를 변경해 보세요."
- 0 < total < 5: "제한된 결과입니다. 검색 범위를 확대하시겠어요?"
- 5 <= total < 20: "적당한 양의 결과를 찾았습니다."
- total >= 20: "많은 관련 항목을 찾았습니다. 상단 결과를 먼저 확인하세요."

### 규칙 3: 도메인 다양성 기반 근거
- 단일 도메인 지배: "주로 [도메인] 관련 결과입니다."
- 다중 도메인: "다양한 카테고리 검색 결과를 혼합했습니다."

## C. 신뢰도 점수 개선 (confidence)
### 기존 규칙 (2단계)
- total 기반: 0 < total >= 10 → high(0.7), 0 < total < 10 → medium(0.5), total=0 → low(0.2)

### 신규 규칙 (3단계)
1. **결과 다양성 (diversity_factor)**
   - 도메인 다양성: (도메인 종류 / 전체 가능 도메인) * 0.3
   - 타입 다양성: (타입 종류 / 전체 가능 타입) * 0.2
   - 예: 도메인 2종류, 타입 2종류 → 0.1 + 0.1 = 0.2

2. **결과 품질 (quality_factor)**
   - 정확 매칭 비율: 상위 3개 중 완전 일치 수 / 3 * 0.2
   - 예: 3개 모두 정확 일치 → 0.2

3. **점수 계산**
   ```
   base_score = min(0.7, 0.2 + (total / 20 * 0.5))
   diversity_score = diversity_factor
   quality_score = quality_factor
   final_score = min(0.95, base_score + diversity_score + quality_score)
   ```

4. **신뢰도 등급**
   - score >= 0.75 → 'high'
   - 0.45 <= score < 0.75 → 'medium'
   - score < 0.45 → 'low'

## D. 캐싱 최적화 (선택사항, 4주 이후)
- 동일 쿼리 1시간 내 재검색 시 briefing 캐싱
- Redis 미 사용 환경: 메모리 딕셔너리 + TTL 관리

## E. 구현 및 테스트
### 구현 파일
- `scripts/ai_briefing/stage3_rules.py`
  - `DomainRuleMap`: 도메인 키워드 → 필터 매핑
  - `TypeRuleMap`: 타입별 다양화 규칙
  - `QueryAnalyzer`: 쿼리 특성 분석
  - `ResultAnalyzer`: 결과 특성 분석
  - `ConfidenceCalculator`: 신뢰도 점수 계산
  - `BriefingBuilder`: 근거 문장 생성

- `tests/test_unified_search_briefing_stage3.py`
  - 추천 필터 규칙 테스트 (도메인별, 타입별)
  - 근거 문장 생성 테스트 (쿼리/결과 기반)
  - 신뢰도 점수 테스트 (다양성/품질 계산)
  - 엣지 케이스 (total=0, 완전 일치, 등)

### 성능 목표
- 규칙 계산 추가 지연: < 50ms
- briefing 누적 지연: p95 < 200ms (네트워크 포함)
